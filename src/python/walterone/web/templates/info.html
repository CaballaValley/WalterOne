WalterOne

This is a server batle game, the goal is win in a battle in the midle of a map.

Main Concepts

Here we have the concepts about the game in general.

User

This is the user in the system with a password to can do a login.

IA

It's an entity related with a user and the matches to can play.

Match

It's an entity of a game where the IA's selected can play.

Match IA

It's an entity with a id for match/IA

Map

Every match has a map where the IA's can move throught the zones in the map.

Zone

Every map has multiple zones connected between then, like a undirected graph.

Triggers

the zones could have enable 3 diferents events that the IA get for this match when go in the zone, lucky_unlucky, go_ryu, karin_gift.

Go Ryu

When a IA get this trigger the next attack increment the damage between 40-50% but increment the damage received in the next attack on the same percentage, once time used you lost the trigger.

Lucky Unlucky

While a IA has enable this trigger, you have an 80% to avoid all the damage from an attack and has only a 20% to do an attack, when you lost all the charges the triggers is disable.

Karin gift

When you go on a zone where this trigger is enable, you recover between 20-30% of your MAX life.

API REST Endpoints

These are the endpoints to comunicate with the server throught the API REST, all the endpoints needs a login session active.

Find(http://SERVER_HOST:PORT/find/?match=MATCH_ID)(GET)

Endpoint to get information about users and triggers include the actual zone of the IA in the match.

Attack(http://SERVER_HOST:PORT/attack/)(POST)

Endpoint to attack another user that is in the same zone with you, you no need to send two parameters match=the id of the match, attack_to=the match_id of the other user, this requests has a retention period of 2 seconds.

Defends(http://SERVER_HOST:PORT/defends/)

Endpoint to enable/disable defend mode to avoid the 25% of the damage received, you no need to send two parameters active=boolean with the status of the defend, match_ia=your match_ia id.

Move(http://SERVER_HOST:PORT/moves/)

Endpoint to move from your actual zone to a zone connected, you no need to send two parameters to_zone=destiny zone_id, match=match id, , this requests has a retention period of 2 seconds.
